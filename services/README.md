# Services 目录说明

该目录用于存放独立服务/网关子工程。

## 当前子目录

- `gateway-api/`：网关 API（服务入口）
- `mcp-gateway/`：MCP 网关相关实现

## 约定

1. 每个子服务应在其目录内维护最小运行说明（启动命令、依赖、配置）。
2. 服务专属配置与脚本应留在各自子目录，不回流到仓库根目录。
3. 服务运行时缓存、构建输出应被忽略，不纳入版本控制。

## 后续治理建议

- 统一服务健康检查与日志字段规范，便于跨服务排障。
